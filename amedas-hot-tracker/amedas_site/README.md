# 全国アメダス高温予測サイト

「今日はどこまで気温が上がるのか」をリアルタイムで監視・予測する全国版サイト。
Flask製。スマホ最優先のレスポンシブデザイン。

## ローカルで動かす

```bash
python -m venv venv
source venv/bin/activate   # Windowsは venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

ブラウザで http://localhost:5000 を開く。

デフォルトは **モックモード**（AMEDAS_MOCK=1）で動作し、ネットワークに一切アクセスせず
気温推移をその場で生成します。ネットワークが使える環境では気象庁の観測点マスタ
（約1,300地点）を自動取得してそのまま使い、取得できない場合は内蔵の主要59地点に
自動フォールバックします（`amedas_data.get_full_station_master()`）。
まずはこのモードでUI・機能全体を確認できます。

## 実データ（気象庁アメダス）で動かす

```bash
export AMEDAS_MOCK=0
python app.py
```

`amedas_data.py` 内で気象庁の公開JSONエンドポイント（station一覧・最新観測・
地点別当日推移）へアクセスします。エンドポイントの仕様は気象庁側の都合で
変更される可能性があるため、`amedas_data.py` の `STATION_LIST_URL` /
`LATEST_TIME_URL` / `LATEST_MAP_URL_TMPL` / `POINT_URL_TMPL` を必要に応じて
調整してください。

実データモードでは以下の点にも配慮しています。

* 全国ランキング等で毎回全地点の時系列を取得するとAPI負荷が大きいため、
  現在気温の上位40地点のみ地点別時系列を取得して予測計算を行います
  （`app.py` の `TOP_N`）。
* 観測点マスタは `get_full_station_master()` が気象庁から自動取得し、
  24時間分をディスクキャッシュします（`.station_cache.json`、gitignore推奨）。
  都道府県は47都道府県庁所在地への最近傍判定、平年値・記録的高温は
  内蔵59地点からの逆距離加重(IDW)補間による**概算値**です。地点詳細ページの
  記録比較セクションには、概算値を使っている地点にその旨の注記が表示されます。
  正式な統計値を使う場合は気象庁「過去の気象データ・検索」等から
  別途データベース化して差し替えてください。

## Renderへのデプロイ

このリポジトリには `render.yaml` を同梱しています。

1. GitHubにpushする
2. Render ダッシュボードで「New +」→「Blueprint」を選び、このリポジトリを指定
3. 環境変数 `AMEDAS_MOCK` を `1`（デモ）または `0`（実データ）に設定
4. デプロイ完了後、発行されたURLでアクセス

手動でWebサービスを作る場合:

* Build Command: `pip install -r requirements.txt`
* Start Command: `gunicorn app:app --bind 0.0.0.0:$PORT`

## ディレクトリ構成

```
app.py               Flaskルーティング・データ集約・検索/比較API・gzip圧縮
amedas_data.py        観測点マスタ(全国約1,300地点)・実データ取得・モックデータ生成
predictor.py           推定最高気温・40℃到達確率・類似日補正などの予測ロジック
templates/            index / station / compare / search の各ページ
static/css/style.css   スマホ最優先のスタイル（タップ領域・スケルトン表示対応）
static/js/             ランキング・マップ・グラフ・比較・検索のフロント処理
render.yaml            Render.com用デプロイ設定
```

## 実装済み機能

* **全国対応**：気象庁の観測点マスタを自動取得し、全国約1,300地点に対応
  （取得不可時は内蔵59地点にフォールバック）。`/api/search` `/api/stations/master`
  `/api/prefs` により、地点検索・比較ページの候補一覧を軽量・高速に提供。
* トップページ：全国マップ（色分け・タップで詳細へ）、5種類のランキング
* 地点詳細ページ：現在気温／今日の最高／推定最高気温（予測幅つき）／
  40℃到達確率／記録更新確率／今月の最高到達確率／ピーク予想時刻／
  昇温速度解析（30分・1時間変化、類似日補正、気象補正係数）／気象要素／
  高層気温（模式）／記録比較（歴代・今年・今月目安・平年・全国＆都道府県順位）
* 予測ロジック：昇温ペースの回帰推定に加え、湿度・風速・日照による補正係数、
  簡易的な類似日補正、信頼区間(σ)の動的な広狭、40℃/記録/月最高到達確率を算出
* グラフ：本日の推移／推定曲線／昨日の推移（ON/OFF）／平年値（ON/OFF）
* 検索：サーバー側 `/api/search` によるインクリメンタル検索（デバウンス200ms）、
  都道府県チップでの絞り込み
* 比較：検索して追加する方式で最大4地点を選択し、テーブル・グラフ重ね表示・
  全国／都道府県順位を並べて比較
* UI：タップ領域の拡大、読み込み中のスケルトン表示、gzip圧縮・静的ファイル
  キャッシュによる読み込み高速化
* 自動更新：90秒ごとにフロントで再取得し、フッターに最終更新時刻を表示

## 既知の制約・今後の改善余地

* 都道府県判定は都道府県庁所在地への最近傍判定のため、離島など一部地点で
  実際の所属都道府県と異なる場合があります。
* 平年値・記録的高温気温は内蔵59地点からのIDW補間による概算値です
  （`station.estimated_values` が `true` の地点）。正式統計値への差し替えを推奨します。
* 過去の類似日表示、ヒートマップアニメーション、CSVダウンロード、SNS共有、
  ダークモード、プッシュ通知などは、今回のAPI構成
  （`/api/station/<id>` 等）を土台に追加しやすい設計にしています。
